DISCOVERED ISSUES:
  ETL WASM ABI detail: _start() does not return a value to the JS caller;
  instead it calls proc_exit(main_return_value) via the WASI import. The
  Node.js runner must capture the value from the proc_exit stub, not from the
  return of _start(). This is now correctly handled in calculator_runner.js and
  documented in calculator.html.
