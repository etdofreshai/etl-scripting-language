  Shipped the file_transform CLI example (examples/cli/file_transform.etl, ~37 lines). WHAT WAS LEFT UNDONE:
  - etl_argv_copy (returns i32) was added to runtime but only the void wrapper etl_argv_get is used from ETL, since ETL compiler does not allow discarding non-void return values from expression statements.
  - Buffer cap is 4096 bytes; files larger than that are silently truncated. Acceptable given the 50-line ETL constraint.
DISCOVERED ISSUES:
  - ETL compiler (compiler0) forbids expression statements that call non-void extern functions; callers must assign the return value or use a void wrapper.
  - ETL's indexed array assignment requires the RHS to be the same element type (i8); arithmetic on i8 values yields i32, so uppercase conversion required a C-side helper rather than inline ETL.
  - Generated C main is main(void); argv must be accessed via /proc/self/cmdline since no argc/argv is passed through the generated entry point.
