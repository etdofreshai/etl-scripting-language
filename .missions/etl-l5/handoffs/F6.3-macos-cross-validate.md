WHAT WAS LEFT UNDONE:

DISCOVERED ISSUES:
  zig's bundled clang is stricter than gcc on macOS targets: it triggers
  -Wparentheses-equality warnings (from compiler0-generated C double-parentheses
  patterns) that are fatal by default. Resolution: use -Wno-parentheses
  -Wno-unused-variable -Wno-unused-parameter instead of -Wall -Wextra for the
  macOS cross-compile step. Documented in the script header.
  The `file` utility is absent on this Linux host; python3 magic-byte inspection
  is used as a portable substitute in both scripts.
